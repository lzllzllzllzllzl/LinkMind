# GitHub 代码推送指南

> 还在为如何把代码上传到 GitHub 发愁吗？这份指南将带你轻松掌握两种推送方式！

本指南将介绍两种将本地代码推送到 GitHub 的方法：使用终端命令行和使用 VS Code 图形界面。

---

## 一、使用终端（命令行）推送

### 1. 初始化本地仓库（如未初始化）

如果你的项目还不是 Git 仓库，需要先初始化：

```bash
git init
```

### 2. 添加远程仓库地址

将本地仓库与 GitHub 仓库关联（将 `your-repo-url` 替换为你的仓库地址）：

```bash
git remote add origin your-repo-url
```

查看远程仓库是否添加成功：

```bash
git remote -v
```

### 3. 暂存更改

将所有文件添加到暂存区：

```bash
git add .
```

或者添加特定文件：

```bash
git add 文件名
```

### 4. 提交更改

填写提交信息，描述这次修改的内容：

```bash
git commit -m "提交描述信息"
```

### 5. 推送到 GitHub

将代码推送到远程仓库：

```bash
git push origin main
```

如果是第一次推送，可能需要使用 `-u` 参数设置上游分支：

```bash
git push -u origin main
```

---

## 二、认证方式说明

### HTTPS 方式
- 每次推送时需要输入用户名和 **Personal Access Token (PAT)**
- 可以在 GitHub 设置 → Developer settings → Personal access tokens 生成
- 优点：无需配置 SSH 密钥

### SSH 方式
- 需要先配置 SSH 密钥
- 生成密钥对：`ssh-keygen -t ed25519 -C "your-email@example.com"`
- 将公钥添加到 GitHub 账户
- 优点：推送时无需每次输入用户名和密码

## 三、使用 VS Code 图形化推送

### 1. 打开源代码管理视图

点击 VS Code 左侧的 **源代码管理** 图标（通常显示为分支形状）。

### 2. 暂存更改

- 在"更改"区域右键点击文件，选择 **暂存更改**
- 或者点击文件旁边的 **+** 图标暂存单个文件
- 也可以点击 **暂存所有更改** 按钮一次性暂存所有文件

### 3. 填写提交信息

在提交框中输入本次修改的描述信息。

### 4. 提交更改

点击提交按钮（**√** 图标）提交更改。

### 5. 推送代码

- 点击 **同步更改** 按钮（带有循环箭头的图标）
- 或者点击 **推送** 按钮（向上的箭头）单独推送

> ⚠️ **注意**：首次推送可能需要登录 GitHub 账号或输入 Token（如果使用 HTTPS 方式）。

---

## 注意事项

⚠️ **删除本地文件不会影响 GitHub 上已推送的内容**

本地删除文件后，需要：
1. 暂存删除操作：`git add .`
2. 提交删除：`git commit -m "删除文件"`
3. 推送到远程：`git push`

这样 GitHub 上的文件才会被删除。
